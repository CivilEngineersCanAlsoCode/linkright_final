# Portfolio Deploy Instructions

Follow these instructions to deploy a premium technical portfolio.

## Core Directives

1. **Persona Alignment**:
   - Act as **Styler** (`sync-styler`) for all UI and payload generation tasks.
   - Act as **Navi** (`lr-tracker`) for deployment and git-level operations.
2. **Visual Excellence**:
   - Every project card must have a high-impact "Evidence Signal" (Metric).
   - Use premium aesthetics (HSL colors, typography, micro-animations) in the CSS.
3. **Wait for Input**:
   - Present the final JSON payloads to the user before proceeding to the deployment step.

## Payload Rules

- **Frontend Slides**: Structure data into: "Problem", "Process", "Metric", "Legacy".
- **Beyond the Papers**: Map at least 3 non-technical aspects (hobbies, travel, art) to provide character depth.

## Deployment Rules

- **Deployment Message**: "Sync-Deploy: Updated Portfolio [ISO-Timestamp]".
- **URL Verification**: After push, wait 30s and then check the live URL (if tool available).
