# Phase G: Gap Analysis - Categorization

## MANDATORY EXECUTION RULES (READ FIRST)
✅ ALWAYS categorize gaps for targeted strategies.
🛑 NEVER group dissimilar gaps into same category.

## DEPENDENCIES
- Requires: `step-49-baseline-gaps` output (gaps_analysis.yaml)

## EXECUTION PROTOCOLS
1. [READ] Load gaps_analysis.yaml from previous step.
2. [CATEGORIZE] Bucket gaps by remediation strategy:
   - Fillable gaps: Can quickly learn/demonstrate
   - Positioning gaps: Reframe existing skills to fit
   - Compensation gaps: Offer alternative strengths
   - Negotiation gaps: Accept as risk but address in conversation
   - Dealbreaker gaps: May fail position (flag for review)
3. [DEVELOP] For each category, identify remediation approach:
   - Fillable: Create content showing learning initiative
   - Positioning: Narrative to reframe existing strengths
   - Compensation: Highlight alternative valuable skills
   - Negotiation: Prepare discussion points
4. [PRIORITIZE] Rank gaps by remediation timeline and effort.
5. [OUTPUT] Generate gap_strategy.yaml with remediation plans.

## OUTPUT
- gap_strategy.yaml: Categorized gaps with remediation strategies
- Next step: step-51-inquisitor-prep

## NOTES
- Gap categorization drives inquisitor question generation
- Strategy determines content focus in Phase J
- Dealbreaker gaps require special handling
