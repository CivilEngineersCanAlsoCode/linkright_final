# Phase I: Narrative Mapping - Story Structure

## MANDATORY EXECUTION RULES (READ FIRST)
✅ ALWAYS structure narrative around validated signals.
🛑 NEVER create fictional narratives without evidence.

## DEPENDENCIES
- Requires: `step-47-signal-validate` output (signals_final_validated.yaml)
- Requires: Inquisitor dialogue responses (step-51 phase H)

## EXECUTION PROTOCOLS
1. [READ] Load validated signals and inquisitor responses.
2. [STRUCTURE] Create narrative arc with 3-4 story sections:
   - Origin: How user entered this field (background + early signals)
   - Growth: Key learning curves and capability development
   - Mastery: Evidence of expertise and notable achievements
   - Vision: Future direction and continued growth
3. [MAP] Assign top-ranked signals to each narrative section.
4. [FLOW] Ensure narrative flow is logical and compelling.
5. [OUTPUT] Generate narrative_structure.yaml with story outline.

## OUTPUT
- narrative_structure.yaml: Narrative arc with signal mapping
- Next step: step-53-content-draft

## NOTES
- Narrative structure guides content writing
- Each section anchored to evidence (signals)
- Structure should feel natural, not forced
