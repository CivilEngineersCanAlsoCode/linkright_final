# Phase J: Content Writing - Bullet Drafting

## MANDATORY EXECUTION RULES (READ FIRST)
✅ ALWAYS draft with evidence from signals.
🛑 NEVER use vague or unquantified claims.

## DEPENDENCIES
- Requires: `step-52-narrative-struct` output (narrative_structure.yaml)

## EXECUTION PROTOCOLS
1. [READ] Load narrative_structure.yaml and signal evidence.
2. [DRAFT] For each narrative section, create 3-5 positioning bullets:
   - Start with action verb (Designed, Built, Led, etc.)
   - Include specific context (scope, scale, impact)
   - Quantify where possible (%, time saved, users impacted)
   - Land JD-aligned keywords naturally
   - Format: One sentence, strong, memorable
3. [REVIEW] Check bullets for:
   - Evidence-backing (each claims must have signal evidence)
   - JD alignment (keywords, requirements, value match)
   - Impact clarity (reader understands value delivered)
4. [COMPILE] Organize bullets by narrative section.
5. [OUTPUT] Generate content_bullets_draft.yaml with full text.

## OUTPUT
- content_bullets_draft.yaml: Positioned content bullets with evidence
- Next step: step-54-layout-check

## NOTES
- Content bullets form the core of positioning
- Each bullet must be specific and evidence-backed
- Quantification dramatically increases credibility
