# Phase M: Final Scoring - Quality Metrics

## MANDATORY EXECUTION RULES (READ FIRST)
✅ ALWAYS measure against success criteria systematically.
🛑 NEVER skip quality validation.

## DEPENDENCIES
- Requires: `step-55-styling` output (styled_portfolio.html)
- Requires: Success criteria from phase kickoff

## EXECUTION PROTOCOLS
1. [READ] Load portfolio and success criteria checklist.
2. [MEASURE] Assess portfolio against 10+ quality dimensions:
   - Content accuracy (no factual errors)
   - JD alignment (keywords present, requirements addressed)
   - Evidence quality (claims backed by specific examples)
   - Writing quality (clarity, conciseness, impact)
   - Visual design (professionalism, brand fit)
   - Completeness (all key achievements covered)
   - Positioning clarity (reader understands value)
   - Format compliance (one-page length, readable)
   - Gap coverage (addressed major gaps identified in Phase G)
   - Impact projection (interviewer will want to call)
3. [SCORE] Generate quality_metrics.yaml with all scores.
4. [FLAG] Identify any dimension scoring below 70%.
5. [OUTPUT] Generate final_quality_report.md with detailed findings.

## OUTPUT
- quality_metrics.yaml: Scored dimensions
- final_quality_report.md: Comprehensive quality assessment
- Next step: Delivery and deployment

## NOTES
- This is final gate before delivery
- Flagged dimensions trigger rework before release
- Quality score predicts success likelihood
