# Phase J (Extended): Content Refinement - Positioning Polish

## MANDATORY EXECUTION RULES (READ FIRST)
✅ ALWAYS refine content for maximum impact.
🛑 NEVER leave weak or vague language.

## DEPENDENCIES
- Requires: `step-53-content-draft` output (content_bullets_draft.yaml)
- Requires: `step-60-narrative-refine` output (narrative_refined.md)

## EXECUTION PROTOCOLS
1. [READ] Load drafted content and refined narrative.
2. [ENHANCE] Improve each bullet:
   - Strengthen action verbs (did → championed, designed, engineered)
   - Add missing quantifiers if available (reduced from 2h to 5min)
   - Integrate narrative story elements where relevant
   - Check for JD keyword alignment
3. [REVIEW] Assess each bullet for impact:
   - Would this make someone want to interview?
   - Is the achievement clear and specific?
   - Does it differentiate from typical applicant?
4. [ITERATE] Revise weak bullets until strong.
5. [OUTPUT] Generate content_refined.yaml with polished bullets.

## OUTPUT
- content_refined.yaml: Polished, impact-focused content
- Next step: Layout validation

## NOTES
- Great bullets are specific, quantified, and differentiated
- Each bullet should make reader curious to learn more
- Avoid generic language (experienced, knowledgeable, etc.)
