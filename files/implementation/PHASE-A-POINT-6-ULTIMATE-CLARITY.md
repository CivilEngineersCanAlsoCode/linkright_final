## Sub-Question 1: FORMAT — What Type of File?

### The Situation

After Phases D-M complete, you'll have a **finished resume** that's been:
- ✅ Analyzed against the JD
- ✅ Optimized with your signals
- ✅ Styled with company colors
- ✅ Validated for A4 page fit

Now the system asks: "In what format should I save this for you?"

### The Options

**Option A: PDF Only**

```
What is PDF?
  → A file format (Portable Document Format)
  → Looks the same on every computer
  → Can be printed to paper
  → Most professional, formal format

Filename example:
  Google_Satvik_Jain_Resume_2026-03-09.pdf

Size: ~1.5 MB

How to use it:
  ✓ Email to recruiter as attachment
  ✓ Upload to job board (LinkedIn, Indeed, etc.)
  ✓ Print to paper for in-person interview
  ✓ Use as "official" version

When to choose PDF only?
  → You're applying to formal companies (Google, Amazon, McKinsey)
  → You're uploading to job boards
  → You want 100% guarantee of formatting consistency
  → You don't need interactive features
```

**Option B: HTML Only**

```
What is HTML?
  → A web page format
  → Interactive, clickable, responsive
  → Material 3 design system (pretty UI)
  → Can be opened in any web browser

Filename example:
  Google_Satvik_Jain_Portfolio_2026-03-09.html

Size: ~500 KB

How to use it:
  ✓ Share via direct link to recruiter
  ✓ Embed in personal website
  ✓ Share via email (attach or link)
  ✓ Show off Material 3 design
  ✓ Include navigation, interactive elements

When to choose HTML only?
  → You're sharing with tech-savvy recruiter
  → You want to showcase design/interactivity
  → You're applying to startups (less formal)
  → You want a portfolio, not just resume
```

**Option C: Both (Recommended) ⭐**

```
What is Both?
  → Generate BOTH PDF AND HTML files
  → Two separate files, both complete
  → One formal (PDF), one interactive (HTML)
  → Maximum flexibility

Filenames example:
  Google_Satvik_Jain_Resume_2026-03-09.pdf
  Google_Satvik_Jain_Portfolio_2026-03-09.html

Total size: ~2 MB

How to use it:
  ✓ Send PDF to formal companies
  ✓ Send HTML link to tech companies
  ✓ Use whichever is appropriate for context
  ✓ Have backup if one doesn't work

When to choose Both?
  → You don't know the company's preference yet
  → You're applying to mix of companies (some formal, some tech)
  → You want maximum flexibility
  → You want to test both formats
```

### Your Decision for Sub-Question 1

```
System asks: "What format do you want?"

Your options:
  A. PDF only        → For formal submissions
  B. HTML only       → For interactive showcase
  C. Both (PDF+HTML) → Best option (most flexible)

Recommended: Choose C (Both)
  Why? You generate once, use format that fits each company
```

## Sub-Question 3: FILENAMES — How Should Files Be Named?

### The Situation

System asks: "How should I name the files?"

### The Answer (Simple)

**You DON'T decide filenames. System auto-generates them.**

```
System uses this pattern:
  [CompanyName]_[YourName]_[Type]_[Date].[ext]

Breakdown:
  [CompanyName]  → Company you're applying to (e.g., "Google")
  [YourName]     → Your full name (hardcoded as "Satvik Jain")
  [Type]         → File type (Resume or Portfolio)
  [Date]         → Today's date (e.g., 2026-03-09)
  [ext]          → File extension (.pdf or .html)

Examples:

If you chose Format = PDF:
  Google_Satvik_Jain_Resume_2026-03-09.pdf

If you chose Format = HTML:
  Google_Satvik_Jain_Portfolio_2026-03-09.html

If you chose Format = Both:
  Google_Satvik_Jain_Resume_2026-03-09.pdf
  Google_Satvik_Jain_Portfolio_2026-03-09.html

If you apply to Spotify on same day:
  Spotify_Satvik_Jain_Resume_2026-03-09.pdf
  Spotify_Satvik_Jain_Portfolio_2026-03-09.html

If you apply to Google again on different day (2026-03-15):
  Google_Satvik_Jain_Resume_2026-03-15.pdf
  Google_Satvik_Jain_Portfolio_2026-03-15.html
  (Different date = different version)

Why this pattern?
  ✅ Company name first → Easy alphabetical sorting by company
  ✅ Your name → Clearly whose resume it is
  ✅ Type indicator → Quick visual (Resume vs Portfolio)
  ✅ Date included → Track which version, when created
  ✅ Professional format → Looks good when emailing
  ✅ No ambiguity → Same company, different dates = tracked
```

### Your Decision for Sub-Question 3

```
System asks: "What filename pattern?"

Your options:
  A. Default pattern  → Auto-generated by system ← ONLY OPTION

Recommended: Accept default
  Why? Professional, unambiguous, tracks everything
```

## Common Misconceptions (Cleared)

### Misconception 1: "Do I have to decide all 3 sub-questions?"
**Answer:** YES. All 3 are required:
- Sub-Q1 (Format): Mandatory
- Sub-Q2 (Location): Mandatory
- Sub-Q3 (Filenames): Automatic (you just accept)

### Misconception 2: "Can I change Point 6 decisions later?"
**Answer:** NO (in one session). But:
- You can run Phase A multiple times for different companies
- Each run saves to different folder
- Example:
  - Session 1: Google → Vault/Google/
  - Session 2: Spotify → Vault/Spotify/
- Both organized separately

### Misconception 3: "If I choose PDF, can I also get HTML?"
**Answer:** YES! Choose "Both" in Sub-Question 1
- You get BOTH files automatically
- Same session, both files generated
- No extra work or time

### Misconception 4: "Do I manually create the folder structure?"
**Answer:** NO. System creates automatically:
- You choose Vault location
- System creates: Resumes/Google/2026-03-09/ automatically
- If folder exists, uses existing
- If folder doesn't exist, creates it
- You don't touch filesystem

### Misconception 5: "If I apply to Google twice, will files overwrite?"
**Answer:** NO. Date tracking prevents conflicts:
- Application 1 (2026-03-09): Google_Satvik_Jain_Resume_2026-03-09.pdf
- Application 2 (2026-03-15): Google_Satvik_Jain_Resume_2026-03-15.pdf
- Different dates = different files
- Both preserved in vault

### Misconception 6: "Is Point 6 only for output location?"
**Answer:** NO. Point 6 is FORMAT + LOCATION + FILENAMES:
- Format: PDF vs HTML (output type)
- Location: Vault vs Desktop (where to save)
- Filenames: Pattern used (how to name)
- ALL three together = Point 6
